変更後は必ず `git add -A && git commit && git push`（ユーザーが求めたら）。GitHub Pages が自動反映する。

## デザインの世界観（壊さないこと）

- **水彩・手描きの温かさ**が最重要。クリニカル／コーポレートな見た目にしない
- 基調クリーム: `#f6f0e1`（全セクション共通の地色。新セクションもこの色に）
- カード地: `#fffdf7`、枠: `rgba(120,95,55,0.10)`、影: `0 18px 36px -22px rgba(120,95,55,0.38)`
- インク: `--w-ink: #2f4a3a`／soft: `#6b8877`、アクセント緑 `#7cb342`、水色 `#4fb3c9`、テラコッタ eyebrow `#c98a4f`
- 見出しフォント: Zen Maru Gothic（`--w-serif`）。見出し29px／本文13.5px前後
- 見出しの1〜2語だけ色を変える `.accent-char`（cyan）/`.warm`（緑）/`.pop`（ピンク）を踏襲

## セクションのつなぎ目（シーム）— 重要

水彩画像セクション（letter/features/day/fee/visit/careplan）は:
1. 親セクションに `background-color: #f6f0e1`
2. 画像に **Sカーブの mask-image フェード**（上72px・下44px、visit-flowのみ下20px — 画像下端のリボン文字を食わないため）
3. `margin-top: -30px` で隣とクロスフェード

**フェードを長くしすぎると画像内の文字が消える**（過去に発生）。触るときは必ず全シームを目視確認。

## レスポンシブ

- ブレークポイント: 1080px（ページ枠を外す）／760px（モバイルレイアウト）

## 試作由来の演出（本番リリース時に相談して判断）
